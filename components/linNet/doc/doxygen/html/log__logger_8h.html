)

Request access to the output streams for for direct writing using the C stdio library functions.

Complex formatted ouput might be easier using the output streams directly. This function reurns the handles of up to two streams: the opened log file and/or the console output. The calling function may use the C stdio functions to write into these streams.
The usage of the streams needs to be finished before the class' own output operations are again used (see ).

Parameters:
hObjThe handle to the logger object to operate on.
phConsoleThe handle of the console output stream is placed in * phConsole. Or NULL if no console output takes place. No action if phConsole is NULL.
phLogFileThe handle of the log file stream is placed in * phLogFile. Or NULL if no log file has been opened. No action if phLogFile is NULL.

Write a single logging message and terminate output with an additional newline.

Write some logging information to the log file and maybe to stdout also and terminate output with an appended newline character.
Do not call this function directly, only use the macros offered in the header file.

Returns:
The number of written characters. If the stdio function return an error this number is not available and this method returns null regardless of possibly written characters.
Parameters:
hObjThe handle to the logger object to operate on.
logLevelThe log level (i.e. importance level) of this message.
formatStringA printf-style format string, which is followed by an argument list of appropriate, variable length.
Remarks:
The log level is use only for reporting it does not control whether to write the logging information or not. This is done by a the function call enclosing conditional phrase, which is hidden in a macro.
This function will append a newline character to the output. A blank line will appear if formatString should end with a newline also.
