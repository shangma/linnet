Split a file-path into path, file name and name extension.

Split a file-path into path, file name and name extension.

Returns
The file name is returned as pointer to the part of input string filePath, which contains the file name (including the extension). No copy of the string is made.
Parameters
pPathIf not NULL, then a malloc allocated string is placed in * pPath, which contains the path (including the final separator char). The caller has to free the string after use.
pPureFileNameIf not NULL, then a malloc allocated string is placed in * pPureFileName, which contains the file name without extension. The caller has to free the string after use.
pExtIf not NULL, then a malloc allocated string is placed in * pExt, which contains the file name extension (including the dot). The caller has to free the string after use.
filePathThe complete file name including the path.
Remarks
The implementation is based on http://stackoverflow.com/questions/1575278/function-to-split-a-filepath-into-path-and-file, visted on Feb 21, 2014

const char* fil_findApplication ( const char *const  appArg0)

Determine the installation directory of this application.

This function tries to find out, where the application linNet is installed. It uses several prioritzed rules to do so:
If the application related environment variable LINNET_HOME is set it is expected to point to the installation directory.
If not, it uses the first program argument to find its own executable. This is not a safe mechanism as the shell might pass the full or a relative path.
In a last step it relates the first program argument to the current working directory and looks for the executable.

Returns
The malloc allocated string that holds the name of the directory, which contains the executable of this application or NULL if the executable could not be localized.
It depends if this is relative or absolute path.
Parameters
appArg0The first argument from the command line.
