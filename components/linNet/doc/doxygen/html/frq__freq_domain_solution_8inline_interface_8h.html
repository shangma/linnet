Detailed Description

Implementation of global inline interface of module frq_freqDomainSolution.c.
All global inline functions are declared "static inline" in order to avoid any problems with undefined or doubly defined symbols at linkage time. The drawback of this declaration is that any client of this module's header file would instantiate its own set of functions. Most clients of this module's header file won't however use the inline functions. By placing the inline interface in a secondary file, any client can decide whether to include the additional header (and instantiate the functions) or not.
