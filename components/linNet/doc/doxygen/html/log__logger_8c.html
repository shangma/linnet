Detailed Description

A simple logging class to write formatted status messages to stdout and into a file.

Create a file logger.

Returns
The Boolean success is returned. You get false if the file couldn't be opened. Logging can be tried even in case of false: Echoing to the console might still work well.
Parameters
phNewLoggerLogger object are used by handles. The handle of the newly created object is returned by reference.
fileNameThe name of the file to write to. Or NULL if no file should be opened.
logLevelThe amount of status information to actually log.
lineFormatThe desired format of the header of each logged line. See enumeration log_lineFormat_t for available values.
initialEchoToConsoleAll logged information is written into stdout also if this is set to true.
doAppendIf true an existing log file is reused and the new status information is appended. The file is deleted and reopened if set to false.

Complex formatted ouput might be easier using the output streams directly. This function reurns the handles of up to two streams: the opened log file and/or the console output. The calling function may use the C stdio functions to write into these streams.
The usage of the streams needs to be finished before the class' own output operations are again used (see ).

Parameters

Write some logging information to the log file and maybe to stdout also.
Normally you won't call this function directly, but only use the macros offered in the header file. These macros make logging conditional by checking the desired level of verbosity.

Returns
The number of written characters. If the stdio function return an error this number is not available and this method returns null regardless of possibly written characters.
Parameters
hObjThe handle to the logger object to operate on.
logLevelThe log level (i.e. importance level) of this message.
formatStringA printf-style format string, which is followed by an argument list of appropriate, variable length.
Remarks
The log level is use only for reporting it does not control whether to write the logging information or not. This is done by a the function call enclosing conditional phrase, which is hidden in a macro.
This function does not append a newline character to the output as boolean log_logLine does.
