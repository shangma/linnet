Detailed Description

The recognized lexical atom is returned by the token stream (function tok_getNextToken) as an object of this structure. The intended use case is to have a single instance of this type and to reuse it throughout the complete tokenizing process. In each call of function tok_getNextToken it'll be filled with the information describing the next lexical atom found in the input stream.
Caution, an object of this type is both, input and output to function tok_getNextToken. If the token is of kind integer or string, then its value is the pointer to a malloc allocated string. This pointer - if not NULL - is freed on entry into the function, in order to make room for the value of the next token. Therefore, prior to the very first invocation of function tok_getNextToken the object needs to be properly initialize. The combination of kind integer or string with a non NULL pointer value would cause a severe error by freeing this seeming heap object. To avoid this problem, always use macro TOK_UNINITIALIZED_TOKEN as a compile time initializer expression for instances of this type.

Field Documentation

tok_tokenType_t tok_token_t::type

The type of the token.

const char* tok_token_t::identifier

If type is tok_tokenTypeIdentifier then value.identifier holds the pointer to a malloc allocated string with the name of the identifier. The caller may copy the pointer value and set identifier to NULL before function tok_getNextToken is invoced the next time. This way the caller would take the ownership of the string and would be in charge to free it after use.

unsigned long tok_token_t::integer

If type is tok_tokenTypeInteger then value.integer holds the value of the (signless) integer literal.

double tok_token_t::fpn

If type is tok_tokenTypeFpn then value.fpn holds the value of the floating point literal.

signed char tok_token_t::character

If type is tok_tokenTypeCharacter then value.character holds the numeric value of a C-style character constant.

const char* tok_token_t::string

If type is tok_tokenTypeString then value.string holds the pointer to a malloc allocated string with the string contents. The caller may copy the pointer value and set string to NULL before function tok_getNextToken is invoced the next time. This way the caller would take the ownership of the string and would be in charge to free it after use.

union { ... } tok_token_t::value

Some tokens are rather token classes: They can have many different values. The found value is returned in this field. Which one to use depends on the token type.
