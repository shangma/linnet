Detailed Description

Implementation of global inline interface of module coe_coefficient.c.
All global inline functions are declared "static inline" in order to avoid any problems with undefined or doubly defined symbols at linkage time. The drawback of this declaration is that any client of this module's header file would instantiate its own set of functions. Most clients of this module's header file won't however use the inline functions. By placing the inline interface in a secondary file, any client can decide whether to include the additional header (and instantiate the functions) or not.

Variable Documentation

mem_hHeap_t coe_hHeapOfCoefAddend

The module exports a globally usable heap for both the coefficients of the LES and the addends of a coefficient.

Remarks
In DEBUG compilation and when the heap is destroyed at application termination time the heap is checked for still allocated objects. The client of the heap should return all requested objects in order to have a memory leak detection in DEBUG compilation.

The module globally accessible heap for both the coefficients of the LES and the addends of a coefficient.

Remarks
Although defined globally, nobody should ever use this variable directly. Instead, use the functional interface declared in coe_coefficient.inlineInterface.h to do so. The inline implementation of this function set demands to declare the memory manager as a global.
